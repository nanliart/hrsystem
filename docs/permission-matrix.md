# HR Permission Matrix

## 1. Scope

This matrix defines the initial permission model for the personnel-ledger MVP.

The base model is:

- role-based capability
- organization-scoped data access
- approval-gated mutations

## 2. Recommended Phase 1 Roles

### group_admin

- Full system administration
- Can manage users, roles, organization hierarchy, and global settings
- Can review all approval requests across the group

### group_hr

- Group-wide HR operator
- Can view all organizations
- Can approve or reject subsidiary change requests
- Can run global exports and summary reports

### subsidiary_hr

- HR operator for one subsidiary or scoped organization tree
- Can view and prepare changes for in-scope employees only
- Cannot directly make official data effective without approval

### read_only_auditor

- Read-only access to approved records and logs within assigned scope
- No mutation, import, approval, or attachment upload capability

## 3. Data Scope Rules

- `group_admin`: full group
- `group_hr`: full group
- `subsidiary_hr`: assigned company or organization subtree only
- `read_only_auditor`: assigned company or organization subtree only

No role should gain out-of-scope access through search, export, attachment links, or imported batch previews.

## 4. Core Capability Matrix

| Capability | group_admin | group_hr | subsidiary_hr | read_only_auditor |
| --- | --- | --- | --- | --- |
| View approved employee records in scope | yes | yes | yes | yes |
| View approved employee records outside scope | yes | yes | no | no |
| Create employee change request | yes | yes | yes | no |
| Edit employee data directly without approval | no | no | no | no |
| Submit education change request | yes | yes | yes | no |
| Submit certificate change request | yes | yes | yes | no |
| Submit attachment change request | yes | yes | yes | no |
| Delete by hard delete | no | no | no | no |
| Mark employee as departed through approval flow | yes | yes | yes | no |
| View departed employee history in scope | yes | yes | yes | yes |

## 5. Approval Matrix

| Capability | group_admin | group_hr | subsidiary_hr | read_only_auditor |
| --- | --- | --- | --- | --- |
| View pending requests for own submissions | yes | yes | yes | no |
| View pending requests across full group | yes | yes | no | no |
| Approve subsidiary requests | yes | yes | no | no |
| Reject subsidiary requests | yes | yes | no | no |
| Approve own request without separation of duties check | restricted | restricted | no | no |
| View approval history in scope | yes | yes | yes | yes |

`restricted` means the product should avoid self-approval by default when an alternative approver exists. If the business insists on emergency override, that action should be explicitly audited.

## 6. Import And Export Matrix

| Capability | group_admin | group_hr | subsidiary_hr | read_only_auditor |
| --- | --- | --- | --- | --- |
| Import workbook for in-scope organization data | yes | yes | yes | no |
| Import workbook affecting out-of-scope organizations | yes | yes | no | no |
| Trigger export for in-scope detail data | yes | yes | yes | no |
| Trigger export for full-group detail data | yes | yes | no | no |
| View summary reports for full group | yes | yes | optional | no |
| Download exported workbook generated by others | yes | yes | scoped only | no |

## 7. Attachment And Sensitive Data Matrix

| Capability | group_admin | group_hr | subsidiary_hr | read_only_auditor |
| --- | --- | --- | --- | --- |
| View masked ID number in scope | yes | yes | yes | yes |
| View full ID number in scope | restricted | restricted | restricted | no |
| Upload certificate attachment in scope | yes | yes | yes | no |
| Download certificate attachment in scope | yes | yes | yes | optional |
| Download archive-location related attachment | yes | yes | restricted | no |
| View archive location field in scope | yes | yes | restricted | optional |

`restricted` means business confirmation is required before enabling broad access. Default product behavior should be the safer setting.

## 8. System Administration Matrix

| Capability | group_admin | group_hr | subsidiary_hr | read_only_auditor |
| --- | --- | --- | --- | --- |
| Manage users | yes | no | no | no |
| Manage roles | yes | no | no | no |
| Manage organization hierarchy | yes | no | no | no |
| Change export template configuration | yes | optional | no | no |
| View audit logs | yes | yes | scoped only | optional |

## 9. Default UI And API Rules

- Hide out-of-scope records from list and search results.
- Re-check scope and role on detail view, download, export, and approval actions.
- Do not rely on front-end filtering alone.
- Keep attachment authorization separate from plain metadata authorization.
- Log denied access attempts for sensitive endpoints when feasible.

## 10. Open Decisions

- Whether `group_hr` and `group_admin` both need access to full ID numbers.
- Whether `read_only_auditor` may download attachments or only preview metadata.
- Whether subsidiary HR may view summary reports beyond their own organization subtree.
- Whether template management belongs only to `group_admin` in phase 1.
